# Post-Exam Assignments

This domain provides a production path from exam mistakes to personalized training tasks.

## Scope
- Error tracking per finished exam attempt.
- Skill-gap aggregation per student and module.
- Actionable assignments generated from weak skills.
- Assignment attempt tracking and completion state.

## Data Model
- `assignment_error_items`
  - One normalized error record per exam source item (`question-*`, `part-*`, `criterion-*`).
  - Deduplicated by `(user_id, exam_kind, exam_id, source_key)`.
- `assignment_skill_gaps`
  - Persistent weak-skill registry per `(user_id, module, skill_key)`.
  - Stores `status`, `severity_score`, `occurrences`, `last_seen_at`.
- `training_assignments`
  - Student-facing training tasks generated from skill gaps.
  - Deduplicated by `(user_id, dedupe_key)`.
- `training_assignment_attempts`
  - Student submissions against assignments with scored feedback.

## Generation Triggers
Assignments are generated by backend on exam lifecycle events:
- Reading submit (finished attempt)
- Listening submit (finished attempt)
- Writing submit and writing result retrieval
- Speaking finalize

Generation is idempotent and safe to re-run.

## API
Base prefix: `/api/v1/assignments`

- `GET /assignments`
  - Query: `limit`, `offset`, optional `module`, optional `status`
  - Returns paged assignment list with latest attempt snapshot.
- `GET /assignments/{assignment_id}`
  - Returns assignment details, linked skill gap, relevant error items, and attempt history.
- `POST /assignments/{assignment_id}/attempts`
  - Body: `{ "response_text": string, "score"?: number }`
  - Creates evaluated attempt, updates assignment status, and updates skill-gap status.

## Notes
- Objective module assignments include accepted answer variants in payload.
- Speaking assignments are criterion-driven (`fluency`, `lexical`, `grammar`, `pronunciation`).
- Writing assignments include low-word-count and low-quality recovery tasks.
